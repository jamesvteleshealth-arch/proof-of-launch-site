Suggested implementation pattern
typescriptimport { Resend } from 'resend';

const resend = new Resend(process.env.RESEND_API_KEY);

export async function submitContact(formData: FormData): Promise<ContactResult> {
  const name = formData.get('name')?.toString() ?? '';
  const email = formData.get('email')?.toString() ?? '';
  const situation = formData.get('situation')?.toString() ?? '';
  const vertical = formData.get('vertical')?.toString() ?? '';
  const platform = formData.get('platform')?.toString() ?? '';
  const message = formData.get('message')?.toString() ?? '';

  // Preserve existing validation

  try {
    const { error } = await resend.emails.send({
      from: 'Proof of Launch <hello@send.proofoflaunch.com>',
      to: 'info@proofoflaunch.com',
      replyTo: email,
      subject: `Proof of Launch — New inquiry from ${name}`,
      html: buildHtmlBody({ name, email, situation, vertical, platform, message }),
      text: buildTextBody({ name, email, situation, vertical, platform, message }),
    });

    if (error) {
      console.error('Resend error:', error);
      return {
        success: false,
        error: 'Email delivery failed, please contact us directly at info@proofoflaunch.com',
      };
    }

    return { success: true };
  } catch (err) {
    console.error('Contact form error:', err);
    return {
      success: false,
      error: 'Something went wrong, please contact us directly at info@proofoflaunch.com',
    };
  }
}
Extract buildHtmlBody and buildTextBody as helper functions so the action stays clean. HTML body should render cleanly — use simple inline styles, table-based layout is fine for email compatibility, no external CSS.
HTML body structure
The email should be scannable at a glance — when James opens a new inquiry, he should be able to see the prospect's name, email, vertical, and situation without scrolling. Suggested structure:
[Header: "New inquiry from [Name]"]

Name: [Name]
Email: [Email]  (mailto link)
Situation: [Situation]
Vertical: [Vertical or "Not specified"]
Platform: [Platform or "Not specified"]

Message:
[Message, preserved with line breaks]

---
Received: [timestamp in human-readable format, e.g., "Apr 23, 2026 at 3:42 PM"]
Reply to this email to respond to [Name] directly.
No branding imagery, no marketing flourishes. This is an internal notification email, not a customer-facing message. Clean and fast-readable.
